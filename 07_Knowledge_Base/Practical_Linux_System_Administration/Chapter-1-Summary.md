# 📖 Chapter 1: The Sysadmin Mindset

**Book:** Practical Linux System Administration (O'Reilly) **Date:** 2026-03-15 **Focus:** Professionalism, Documentation, and Systematic Thinking

## 🧠 Key Concepts

### 1. The Mindset

- **Systems Thinking:** Moving beyond "breaking/fixing" to understanding how a change on a workstation (Leviathan) affects the management cockpit (Ops) and the edge (Core-Router).
    
- **Proactive vs. Reactive:** Building the backup script _before_ moving into complex configurations.
    

### 2. Documentation as a Service

- **The "Why" vs. the "What":** Using Obsidian for intent and Git for version control.
    
- **Learning in Public:** The accountability of a public repo (`Sovereign-Lab-Docs`).
    

### 3. Automation & Efficiency

- **Laziness as a Virtue:** Spending 2 hours automating a 5-minute task so it never has to be remembered again (Crontab + Bash).
