# 🛠️ Runbook: Postfix Mailer Hang Resolution 
**Last Updated:** 2026-03-13 **Target:** 
Proxmox Host / LXCs ## The Problem Proxmox and certain Linux containers attempt to send internal administrative emails (e.g., cron job outputs or sudo warnings) using the default `postfix` mailer daemon. If the daemon is misconfigured or cannot route out, the service hangs, which can cascade into severe system delays or prevent LXCs from shutting down cleanly. ## The Solution We implemented a "Placeholder Fix" for Postfix. ### Action Taken * Reconfigured Postfix to run in `local only` mode to prevent it from attempting DNS lookups or external relays. * Ensured the `postfix` service is either fully functional for local delivery or safely disabled/masked via `systemctl` if internal mail is not required for that specific node. > **Architect's Note:** When deploying new LXCs, verify mailer daemon behavior. If a container hangs on "Stop," check `systemctl status postfix`.
